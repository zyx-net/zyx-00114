数据导出 / 导入

导出完整数据为 JSON 文件，可在其他环境导入恢复。导入时按 ID 去重，已有数据不会被覆盖，不同操作人的草稿和快照不会串线。

🔍 快照审计回放

使用说明

  • 日志导入：选择之前导出的修订日志 JSON 文件导入系统，可与现有日志核对。导入时必须明确操作者身份，审批员可执行冲突重导入。
  • 冲突处理：重复导入同一批日志时，系统会自动识别冲突（ID重复或签名重复），不会静默覆盖，需手动选择策略处理。
  • 审计回放：选择日志ID按时间顺序回放（只读审计，不修改数据），回放记录会区分不同操作者。
  • 快照权限：非草稿 owner 查看快照详情时，正文、保存理由、创建人信息将被脱敏，仅审批员可查看全部。
  • 持久化：导入结果、回放记录、快照可见性均持久化到 db.json，服务重启后保持一致。
